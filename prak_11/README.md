# Практическое занятие №11: Проектирование REST API (CRUD для заметок)

**ФИО:** Булискерия Будимир Бакуриевич  
**Группа:** ПИМО-01-25

Проект демонстрирует проектирование REST API на Go с использованием слоистой архитектуры (handler → service → repository) и реализацией CRUD-операций для сущности "Заметка". В качестве хранилища используется in-memory репозиторий.

## Описание
Данный проект представляет собой backend-приложение для управления заметками с REST API интерфейсом. Реализована слоистая архитектура, разделяющая обработку HTTP-запросов, бизнес-логику и работу с данными. Проект подготовлен для дальнейшей интеграции с базой данных и JWT-аутентификацией.

## Цели работы
1. Освоить принципы проектирования REST API.
2. Научиться разрабатывать структуру backend-приложения на Go.
3. Спроектировать и реализовать CRUD-интерфейс для сущности «Заметка».
4. Освоить применение слоистой архитектуры (handler → service → repository).
5. Подготовить основу для интеграции с базой данных и JWT-аутентификацией.

## Требования
- Go 1.21 и выше

## Установка и запуск

1. Клонируйте репозиторий:
```bash
git clone <ваш-репозиторий>
cd notes-api

2. Инициализируйте проект и установите зависимости:
go mod init example.com/notes-api
go get github.com/go-chi/chi/v5

3. Создайте структуру каталогов согласно схеме ниже.

4. Запустите сервер:
go run ./cmd/api

## API Endpoints

### Создание заметки
curl -X POST http://localhost:8080/api/v1/notes \
  -H "Content-Type: application/json" \
  -d '{"title":"Первая заметка", "content":"Это тест"}'

### Получение всех заметок
curl http://localhost:8080/api/v1/notes

### Получение заметки по ID
curl http://localhost:8080/api/v1/notes/1

### Обновление заметки
curl -X PATCH http://localhost:8080/api/v1/notes/1 \
  -H "Content-Type: application/json" \
  -d '{"title":"Обновлённая заметка", "content":"Новое содержание"}'

### Удаление заметки
curl -X DELETE http://localhost:8080/api/v1/notes/1
